# Custom docs MCP — Claude Code ガイド


### 全体ルール
- 回答が不確かな場合「可能性」などの言葉は使わず決めつけはせずに公式ドキュメント等を参照する
- コンテキスト削減のため必要に応じて積極的にエージェントを活用する
- プロンプトに対する不明点がある場合や課題に直面し別案を実行する場合はユーザーに承認を求める

### 作業前
- 作業対象に応じて AI コンテキストを読む `CONTEXT.md`
- タスク前に関連ファイルを読んでパターンを把握してからコーディング
- ユーザーから質問を受けた場合回答で返し実装はしない
- 実装を行う際は必ず計画を立てユーザーに承認を求める

### 作業中
- 既存ファイルの編集を優先。命令外の不要な抽象化・リファクタリングは行わない
- 型定義を確認してから実装
- `.env` 等環境変数やシークレットをコミットしない
- プロンプトに対し不明点や実行不能な課題に直面し別案を実行する場合は**必ずユーザーに質問を求める**こと

### AI コンテキストの更新
コードを変更した際、内容が変わった場合は対応する ドキュメントも同時に更新すること:
- `CONTEXT.md`
- `README.md`

### 出力
**最終的なユーザーへの出力は必ず日本語で回答すること**
